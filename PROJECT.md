# Hands-on RL Project Definition

## Name
Hands-on RL

## Positioning
A practical reinforcement learning learning work focused on helping learners understand the relationship between algorithms, environments, and training behavior.

## What it is
- a learning-oriented project
- a practical path into DQN and PPO
- a bridge between RL theory, implementation, and experiment interpretation

## What it is not
- not a large RL framework
- not a benchmark zoo
- not a feature-stacking algorithm archive
- not a repo that confuses running code with understanding reinforcement learning

## Core user value
Hands-on RL helps learners:
1. understand why DQN and PPO matter
2. understand why environment design matters in RL
3. move from reading formulas to reading code
4. move from reading code to interpreting training behavior
5. build a reusable mental model of RL experiments

## Current scope
### Included
- DQN
- PPO
- step-by-step implementation structure
- example environments
- learning-path and environment docs
- debug-boundary and experiment-review docs

### Excluded for now
- large algorithm catalog
- benchmark-heavy positioning
- broad environment collection
- optimization for leaderboard performance

## Product thesis
The most valuable RL beginner project is not the one that contains the most algorithms.
It is the one that most clearly connects algorithm design, environment structure, and observed training behavior.

A learner should leave not only with runnable code, but with a more stable mental model of why reinforcement learning behaves the way it does.

## Restructuring thesis
This repository is being reshaped from a teaching code repo into a learning work with:
- clearer positioning
- clearer boundaries
- clearer learner path
- clearer explanation of failure and review
