# What Good Looks Like

A good learner outcome for this project is not just:
- "I ran the script"
- or "I copied the code"

A good outcome is:
- I can explain why DQN and PPO were chosen
- I can explain why the environment matters
- I can point to one likely failure mode in training
- I can interpret a simple reward curve without pure guessing
- I know what I would check first if training looks wrong

That is the standard this project is trying to support.
