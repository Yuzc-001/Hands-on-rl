# Examples as Learning Interfaces

Examples should not be treated as disposable demo scripts.
In this project, examples are learning interfaces.

## Each example should help answer
- why this algorithm is used here
- why this environment is useful here
- what healthy learning should roughly look like
- what common failure signals look like

## Design rule
Every example should be understandable from three angles:
1. algorithm angle
2. environment angle
3. training-behavior angle

If an example only shows code execution, it is incomplete as a learning interface.
