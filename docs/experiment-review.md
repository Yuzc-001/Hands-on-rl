# Experiment Review

Use this after running an example or making an implementation change.

## 1. Task
- What environment was used?
- What algorithm was run?
- What was changed this time?

## 2. Outcome
- Did training improve, fail, stall, or behave inconsistently?

## 3. What happened
- What did the reward curve look like?
- What did loss / exploration / stability look like?

## 4. What changed the result
- Which implementation or configuration detail seems to matter most?

## 5. Boundary / uncertainty
- Is the current problem clearly about implementation, environment, or hyperparameters?
- What is still unclear?

## 6. Reusable lesson
- What should be remembered for the next experiment?

## 7. Next-time rule
- What should be checked first next time?
- What should be changed next, and what should not be changed yet?

## Purpose
The point of an experiment is not only to get a result.
It is to leave behind a better understanding for the next run.
