# Common Mistakes

## 1. Learning algorithms without learning environments
If you do not understand the environment, you will misread the training behavior.

## 2. Running code without knowing what to observe
A script finishing is not the same as understanding what happened.

## 3. Blind hyperparameter tuning
Before changing learning rate or batch size, check the environment interface, data flow, and target computation.

## 4. Treating every noisy curve as failure
Some environments naturally produce noisier learning than others.

## 5. Mistaking more algorithms for better learning
For early learning, depth matters more than catalog size.
DQN and PPO already contain enough structure to teach the most important RL ideas.
